# Mi proceso

Para esta tarea construí una página web utilizando HTML con el objetivo de presentar la visualización y la crónica desarrolladas previamente en el proyecto.

## ¿Cómo lo hice?

Antes de comenzar, revisé las instrucciones de la tarea para identificar los elementos mínimos que debía incluir la página. A partir de eso decidí organizar el contenido en distintas secciones: un encabezado con el título y la bajada, una sección para la visualización, otra para la imagen complementaria, una sección principal para la crónica y un cierre con información adicional y enlaces. Estas secciones fueron escogidas pensando en una nota periodística cualquiera.

Revisé bibliotecas de etiquetas HTML y luego trabajé el archivo en Visual Studio Code. Primero construí la estructura básica del documento incorporando las etiquetas obligatorias que nos pedían.

Luego agregué la crónica y la visualización que había realizado anteriormente. Traté de darle una vuelta a la crónica para que fuese un tanto más completa que la vez anterior, buscando un análisis más profundo del fenómendo del cine categoría R.

Durante el proceso tuve que aprender cómo vincular correctamente imágenes locales dentro de una página web y cómo organizar los archivos para que GitHub Pages pudiera mostrarlos sin errores. Fue un ensayo y error para hacer que la página cumpliera y tuviese lo que se nos pedía.

Una vez terminada la estructura, me dirigí a GitHub Pages para ver cómo se veía. No quedo del todo bonita, pero para una primera ocasión me siento orgulloso.

Algo que me gustaría agregar es la dificultad que supuso comprender cómo se relacionan entre sí los distintos archivos de un sitio web y cómo publicarlos correctamente en internet mediante GitHub Pages.
